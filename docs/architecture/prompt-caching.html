Orion · Prompt Caching — tools / memory / skill / mode 在 4 BP 中的位置(視覺圖解)

Anthropic API 允許每 request 最多 4 個 cache_control breakpoint。Orion 把它們配在 system 兩段 + messages rolling 兩個。本文用 SVG 視覺化解釋 tools / memory / skill / mode prefix / git_status / user 真實輸入 各自落在哪個區、哪個 bp 命中。

BP 1 (static)
BP 2 (session-stable)
BP 3 (rolling read)
BP 4 (rolling write)
永遠固定
每 turn 變

🧮 cache_hit_rate 公式

cache_hit_rate = cache_read_tokens / (cache_read_tokens + input_tokens + cache_creation_tokens)

分母含 cache_creation(本輪 BP 4 寫入),代表「整 prompt 內有多少占比被 cache 省到」。Turn 1 全寫入 → 0%;Turn 2+ 通常 80-90%+;穩定狀態 95%+。
