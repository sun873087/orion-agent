Phase 4 + Phase 5 + Phase 13 + cache 優化(2026-05-10)累積後的實際代碼 • 7 靜態段 • 7 session-stable 段 • 2 per-turn 段 • 4 / 4 cache breakpoints
架構摘要:system list 兩段都享 cache(static 1h / session 1h), messages 用 rolling 2 bp(5m TTL),memory + git_status 改注入 user message 避免破壞 prefix。Anthropic 4 個 bp 已用滿;OpenAI 自動 cache 同一份 prompt 結構即可。

對話累積 — 同一個 Conversation 物件持有 state,user / assistant / tool_use / tool_result 輪流堆疊。compute_message_breakpoints() 每 turn 標 2 個 cache_control:bp #1 = 最後 message(寫入此 turn cache), bp #2 = 倒數第二個 user message(讀上 turn cache)。 長對話 cache hit rate 接近 100%(僅當前 turn 新內容付 1.25× 寫入,過去全部 0.1× 讀取)。
